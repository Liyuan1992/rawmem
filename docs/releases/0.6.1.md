# rawmem 0.6.1

rawmem 0.6.1 completes the hardened capture layer needed by downstream shadow
consumers.

## Added

- Passive Cursor agent-transcript tailer.
- Fictional Claude Code / Codex / Cursor parity fixtures.
- Project/path privacy allowlists and allowed-scope checks.
- Configurable secret redaction before append.
- Artifact policy with `references_only`, `drop`, and `preserve` modes.
- Daemon source coverage, skipped-event, redaction, and tracked-file metrics.
- Doctor checks for source coverage and cursor protocol health.
- Explicit `rawmem rotate --yes` with a new ledger identity and old-ledger
  archive.
- Controlled three-source daemon smoke and a reproducible ledger benchmark.

## Verified scale

On the maintained Windows/Python 3.12 workstation, a fictional 1,000,030-event
606,684,288-byte ledger produced:

- append p50 5.355 ms, p95 6.486 ms;
- cursor batch-of-100 p50 1.016 ms, p95 1.645 ms;
- cursor peak traced memory 274,932 bytes;
- complete chain verification in 8.324 seconds.

The measured append and batch reader use the public locked/sidecar/cursor path;
the benchmark does not load the whole ledger for append or incremental reads.

## Compatibility

`rawmem.event.v1`, `rawmem.cursor.v1`, and `rawmem.event_batch.v1` are unchanged
from 0.6.0. Existing ledgers and cursors remain valid. New privacy settings are
deep-merged over existing config; empty allowlists preserve prior coverage.
