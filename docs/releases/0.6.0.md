# rawmem 0.6.0

rawmem 0.6.0 hardens the append-only ledger for long-running and multi-process
consumers while keeping `rawmem.event.v1` backward compatible.

## Highlights

- OS-backed cross-process append lock.
- Steady-state last-hash sidecar; appends no longer load the full ledger.
- `rawmem verify` with versioned JSON diagnostics.
- `rawmem.cursor.v1` and `rawmem.event_batch.v1` incremental Python API.
- `rawmem export` with atomic cursor-file updates and source/type/project
  filters.
- Explicit ledger-changed, truncated, and invalid-boundary recovery states.

## Upgrade notes

Existing ledgers require no migration. The first 0.6 read or append creates:

```text
events.jsonl.lock
events.jsonl.state.json
```

These are derived coordination files and can be rebuilt. Do not copy a cursor
between ledger files. If a consumer reports `ledger_changed`, `truncated`, or
`invalid`, preserve the old cursor for audit and restart from the returned zero
cursor only after choosing whether to backfill.

rawmem remains local-first, dependency-free at runtime, and does not promote
captured evidence into durable semantic memory.
